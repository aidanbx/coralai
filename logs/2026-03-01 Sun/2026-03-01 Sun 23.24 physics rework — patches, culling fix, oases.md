## What Changed

### 1. Infra maintenance decay
**File:** `experiment.py` (`run_evolution`), `run.py`

Infra now has an optional per-step decay rate. Without energy income, built
infrastructure gradually erodes, making territory expensive to hold. This
creates meaningful cost for over-expansion and pushes organisms to stay near
energy sources.

```python
# Applied in run_evolution before clamp
if self.infra_decay > 0:
    substrate.mem[0, inds.infra] *= (1.0 - self.infra_decay)
```

**CLI:** `--infra-decay 0.002`  (0 = disabled, default)

---

### 2. Infra-as-defense in `explore` kernel
**File:** `physics.py` (`explore` kernel + `explore_physics`), `experiment.py`

The `explore` kernel now raises the colonisation threshold for cells that
have built up infrastructure. An incoming bid must exceed
`cell_energy + cell_infra * defense_coeff` to take a cell. With
`defense_coeff=0`, behaviour is unchanged.

```python
# Before (in Taichi kernel):
max_bid = mem[0, inds.energy, i, j]

# After:
max_bid = mem[0, inds.energy, i, j] + mem[0, inds.infra, i, j] * defense_coeff
```

**CLI:** `--defense-coeff 0.5`  (0 = disabled, default)

---

### 3. Remove day/night energy cycle; patches are the sole energy source
**Files:** `experiment.py` (`run_evolution`), `evolution.py`

The sinusoidal global energy + infra injection (`get_energy_offset`,
`torch.randn * offset`) has been removed entirely. Cells no longer receive
ambient energy. The `PatchesEnvironment` (and `OasesEnvironment`) are now
the only source of energy in the world, which creates real foraging pressure
for the first time.

Removed from `run_evolution`:
- `get_energy_offset` call and `evolver.energy_offset` assignment
- `substrate.mem[0, inds.energy/infra].add_(randn * offset)` noise
- `evolver.energy_offset` display line in `CoralVis`

Removed from `evolution.py`:
- `get_energy_offset` function (dead code)

Energy/infra clamps now floor at `0.0` instead of `0.01`.

### 4. `PatchesEnvironment` — patches with true extinction and birth
**File:** `environments.py`

Replaced the old capacity-regen design (patches never died) with a proper
extinction/birth model:

- Each patch cell holds a finite `max_capacity` energy reserve
- `step()` releases `inject_rate` energy/cell/step and depletes the reserve
- When capacity → 0, the cell goes dark permanently (extinction)
- Every `spawn_interval` steps, new circular patches spawn at random
  locations **only into dead cells** — keeping the total patch count
  roughly stable while cycling their positions

**Usage:**
```bash
python experiments/coral_dev/run.py --env patches
python experiments/coral_dev/run.py --env patches --env-param 12   # 12 patches
```

**Key parameters** (edit class to change; no CLI knobs beyond `--env-param`):
| Param | Default | Meaning |
|---|---|---|
| `n_patches` | 8 | patches at start + per spawn wave |
| `patch_radius_frac` | 0.07 | radius as fraction of min(W,H) |
| `max_capacity` | 30.0 | energy reserve per cell at birth |
| `inject_rate` | 0.06 | energy/cell/step released |
| `spawn_interval` | 250 | steps between spawn waves |

Patch lifespan at defaults: `30 / 0.06 = 500 steps` per cell.

### 5. Fix critical culling bug in `reduce_population_to_threshold`
**File:** `coralai/evolver.py`

**Two off-by-one bugs** caused every cull to scramble the entire simulation:

**Bug A — 1-indexed genome transitions (critical):**
`genome_transitions[old_idx] = len(new_genomes)` was computed *after*
appending, producing values `1, 2, 3…` instead of `0, 1, 2…`. After a
cull, every cell was using the *wrong genome's weights* (shifted by one).
The most-populated genome (which gets new index 0) was running genome 1's
brain; genome 1 was running genome 2's brain, etc.

**Bug B — kill threshold off by one:**
`i > max_population` kept `max_population + 1` survivors. The final
survivor got transition value == `combined_weights.shape[0]`, which
`apply_weights_and_biases` checks as `>= shape[0]` and treats as dead,
zeroing that genome's outputs every step silently.

**Fix:**
```python
# Before (broken):
genome_transitions[index_of_genome] = len(new_genomes)  # 1-indexed!

# After (fixed):
new_idx = len(new_genomes)        # compute BEFORE append → 0-indexed
new_genomes.append(...)
genome_transitions[old_idx] = new_idx

# Kill condition fixed:
if rank >= max_population:   # was: i > max_population
```

Also: `genome_transitions` initialised to `[-1]` (not `[None]`) and
tensor created with `dtype=torch.float32` to match the genome channel.

### 6. `OasesEnvironment` — isolated regions with bridges + patches
**File:** `environments.py`, `run.py`

A grid of `n_cols × n_rows` oases separated by permanently-blacked-out
walls, connected only by narrow bridges. Patches within each oasis are the
sole energy source, seeded relatively small to create strong foraging
pressure within each region.

**Topology construction:**
- Walls punched as full-width strips (x-walls between columns, y-walls
  between rows)
- One bridge per adjacent-oasis pair, placed at the midpoint of the shared
  wall; width = `bridge_width` cells
- Wall–wall intersections (corners) remain permanently blocked
- Mask enforced every step via `_apply_mask`

**Patch management:**
- `patches_per_region` small patches placed randomly inside each oasis at
  seed and at every `spawn_interval` steps
- Patches only spawn into dead cells (`capacity == 0`) and never bleed
  into walls (`* self._mask` guard)
- Each patch cell: `max_capacity=25`, `inject_rate=0.05` → ~500 step
  lifespan

**Usage:**
```bash
python experiments/coral_dev/run.py --env oases
python experiments/coral_dev/run.py --env oases --env-param 4   # 4×4 grid
python experiments/coral_dev/run.py --env oases --env-param 3 --infra-decay 0.002 --defense-coeff 0.5
```

**Key parameters:**
| Param | Default | Meaning |
|---|---|---|
| `n_cols` | 3 | oasis columns (also rows unless `n_rows` set) |
| `wall_width` | 10 | wall thickness in cells |
| `bridge_width` | 4 | bridge opening in cells |
| `patches_per_region` | 2 | patches per oasis |
| `patch_radius_frac` | 0.12 | radius as fraction of min(region_w, region_h) |
| `max_capacity` | 25.0 | energy reserve per patch cell |
| `inject_rate` | 0.05 | energy/cell/step |
| `spawn_interval` | 300 | steps between respawn waves |

```bash
# --- Baseline: just patches, no extra physics pressure ---
python experiments/coral_dev/run.py --env patches

python experiments/coral_dev/run.py --env patches --infra-decay 0.002

# --- Patches + full pressure (decay + defense) ---
python experiments/coral_dev/run.py --env patches --infra-decay 0.002 --defense-coeff 0.5

# --- Oases: 3×3 grid, default settings ---
python experiments/coral_dev/run.py --env oases

# --- Oases: 4×4 grid (smaller oases, more bridges) ---
python experiments/coral_dev/run.py --env oases --env-param 4

# --- Oases: full pressure stack ---
python experiments/coral_dev/run.py --env oases --env-param 3 \
    --infra-decay 0.002 --defense-coeff 0.5

# --- Oases headless, 2000 steps, logged ---
python experiments/coral_dev/run.py --env oases --no-gui --steps 2000

# --- Oases with checkpoint + resume ---
python experiments/coral_dev/run.py --env oases --checkpoint-interval 500
python experiments/coral_dev/run.py --env oases \
    --resume-from runs/coral_dev_TIMESTAMP/checkpoint_0000500 --steps 1000

# --- Profiling / benchmark ---
python experiments/coral_dev/run.py --env oases --no-gui --steps 200 --profile
python experiments/coral_dev/run.py --benchmark --steps 200 --shape 400
```
