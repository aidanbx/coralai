### Softmax → tanh fix (coral_dev/physics.py)

**Before (thesis version):**
```python
inv_liq = mem[0, inds.acts_invest:inds.acts_liquidate + 1]
mem[0, inds.acts_invest:inds.acts_liquidate + 1] = torch.softmax(inv_liq, dim=0)
```
Problem: `softmax(dim=0)` forces `invest + liquidate = 1` per cell. A cell can never "do nothing" — at equilibrium the network wastes maximum energy flip-flopping between invest and liquidate.

**After (coral_dev):**
```python
raw_trade = mem[0, inds.acts_invest] - mem[0, inds.acts_liquidate]
trade = torch.tanh(raw_trade)
mem[0, inds.acts_invest]    = torch.relu(trade)   # ∈ [0, 1]
mem[0, inds.acts_liquidate] = torch.relu(-trade)  # ∈ [0, 1]
```
- Both network outputs influence behavior (their difference determines direction)
- `trade ≈ 0` → both are near zero → cell can genuinely idle
- `trade > 0` → invest only, `trade < 0` → liquidate only — mutually exclusive, not summing to 1
- Magnitude is smooth (tanh), not winner-take-all

The `invest_liquidate` physics is unchanged — it still reads invest and liquidate as rates.

### Priority 1: Checkpointing (HIGH — blocks everything else)

**What's missing**: `SpaceEvolver.save_checkpoint()` is a stub (`pass`). No run is recoverable.

**What's needed**: save enough state to resume a run or analyze it post-hoc.

**Minimal viable checkpoint** (one Python function to fill in):
```python
def save_checkpoint(self, run_dir, step):
    import pickle, json, subprocess
    os.makedirs(run_dir, exist_ok=True)

    # 1. Substrate memory (the entire spatial state)
    torch.save(self.substrate.mem, os.path.join(run_dir, f"substrate_step{step}.pt"))

    # 2. Genome population (NEAT genomes + ages + weights)
    pop_data = {
        "genomes": self.genomes,
        "ages": self.ages,
        "weights": [w.cpu() for w in self.combined_weights],
        "biases":  [b.cpu() for b in self.combined_biases],
        "timestep": self.timestep,
    }
    with open(os.path.join(run_dir, f"population_step{step}.pkl"), "wb") as f:
        pickle.dump(pop_data, f)

    # 3. Run metadata (config, git hash, timestamp)
    meta = {
        "step": step,
        "timestamp": datetime.now().isoformat(),
        "git_hash": subprocess.check_output(["git", "rev-parse", "HEAD"]).decode().strip(),
        "shape": (self.substrate.w, self.substrate.h),
        "n_genomes": len(self.genomes),
    }
    with open(os.path.join(run_dir, "meta.json"), "w") as f:
        json.dump(meta, f, indent=2)
```

**Where to call it**: in the main step loop at `experiments/coral_dev/run.py`, add:
```python
if step % checkpoint_interval == 0 and step > 0:
    evolver.save_checkpoint(run_dir, step)
```

**Run directory convention**: `runs/{experiment}_{timestamp}/` — created at startup, all checkpoints go there.

**Estimated effort**: 1–2 hours. Self-contained, no architectural changes.

### Priority 2: Per-step CSV logging (HIGH — enables analysis without replaying)

**What's missing**: no per-step metrics. Analysis requires re-running everything.

**What to log** (one row per step, negligible cost):
```
step, total_energy, total_infra, n_alive, n_genomes, energy_offset, fps
```

**Implementation**: open a CSV at run start, append one row per step:
```python
import csv
log_path = os.path.join(run_dir, "step_log.csv")
with open(log_path, "w", newline="") as f:
    writer = csv.writer(f)
    writer.writerow(["step", "total_energy", "total_infra", "n_alive", "n_genomes", "fps"])
# In step loop:
    writer.writerow([step, float(tot_e), float(tot_i), alive_n, len(evolver.genomes), fps])
```

**Why this matters**: lets you plot energy dynamics, population diversity, and throughput over time without saving substrate tensors every step. A single CSV per run is trivially archivable.

**Estimated effort**: 30 minutes.

### Priority 3: Culling → Fixed genome pool (MEDIUM — affects biodiversity)

**What's broken**: `reduce_population_to_threshold` kills genomes with the fewest cells when count > 100. This destroys rare-but-not-extinct genomes — exactly the ones that might have just been seeded and need time to establish.

**Fix**: pre-allocate a fixed slot pool (e.g., 256 slots). On genome extinction → slot returns to free list. No arbitrary killing.

```python
# In SpaceEvolver.__init__:
self.pool_size = 256
self.genome_slots = [None] * self.pool_size  # genome or None
self.free_slots = list(range(self.pool_size))

# add_organism_get_key → pop from free_slots, assign slot
# on extinction → detect via cell count == 0, return slot to free_slots
```

The `genome` channel in the substrate uses integer keys — they'd now be slot indices (0..255), stable across the run.

**Impact**: rare genomes persist longer → more biodiversity → different evolutionary dynamics. Also fixes memory growth (pool is bounded).

**Estimated effort**: 2–3 hours. Touches evolver.py only.

### Priority 4: Step-wise complexity metric (MEDIUM — enables comparison)

**What exists**: `analysis/multiscale_complexity.py` (ported from origin/analysis). Computes a renormalization-group complexity scalar on a spatial snapshot.

**What's needed**: wire it into the step loop every N steps (e.g., N=100):
```python
if step % 100 == 0:
    complexity = compute_complexity(substrate.mem[0, inds.energy])
    writer.writerow([..., complexity])  # append to CSV
```

**Why**: this is the primary scientific measurement. It lets you compare: "does the tanh fix produce higher complexity? do spatial resource patches increase complexity?" Without it, comparison is qualitative.

**Estimated effort**: 1–2 hours (mostly understanding the existing API).
