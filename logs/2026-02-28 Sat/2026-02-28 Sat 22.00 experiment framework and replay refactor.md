## What was built

Three structural improvements implemented together:

### 1. `StartEnvironment` base class (`coralai/environment.py`)

Clean class-based abstraction for starting conditions. Before this, environment logic lived as free functions in `env.py` (mask functions, seed functions scattered around). Now:

```python
class StartEnvironment:
    name: str = "flat"
    def seed(self, substrate): pass   # called once at run start
    def step(self, substrate): pass   # called each step if persist=True
    @property
    def persist(self) -> bool: return False
    def to_dict(self) -> dict: return {"name": self.name}
```

Concrete subclasses live in each experiment's `environments.py` so they get snapshot-frozen alongside the physics.

**Environments implemented** (`experiments/coral_dev/environments.py`):
- `FlatEnvironment` — default, no-op
- `HoleEnvironment(radius_frac)` — circular barren center; `persist=True` re-zeros each step
- `RingEnvironment` — same mask as hole
- `StripesEnvironment(n_stripes)` — alternating habitable/barren vertical bands
- `CornersEnvironment(corner_frac)` — four diagonal barren corners

Each caches its mask tensor on first call and zeroes energy+infra in barren zones. Seeds habitable zones with uniform energy/infra.

### 2. `Experiment` base class (`coralai/experiment.py`)

Bundles everything needed to run *and* replay a simulation:

```python
class Experiment:
    name: str
    channels: dict        # Taichi channel definitions
    kernel: list          # spatial offsets
    dir_order: list       # [fwd, left, right] rotation indices
    sense_chs: list       # NN inputs
    act_chs: list         # NN outputs
    neat_config_filename: str

    def make_substrate(shape, device) -> Substrate
    def make_evolver(substrate) -> SpaceEvolver
    def make_env(env_name, param) -> StartEnvironment
    def make_vis(substrate, evolver) -> Visualization
    def run_physics(substrate, evolver)        # abstract
    def run_evolution(substrate, evolver, step) # abstract
```

Key design decision: `_exp_dir = os.path.dirname(os.path.abspath(__file__))` is captured as a class attribute at module load time. This is how `make_evolver` finds `neat.config` when loaded from a snapshot via `importlib` (where `inspect.getfile()` fails on dynamically-loaded modules).

**Why `inspect.getfile` fails**: When Python loads a module via `importlib.util.spec_from_file_location`, torch patches `inspect.getfile` and raises `TypeError: is a built-in class`. The workaround is to capture `__file__` at definition time, not at call time.

`EXPERIMENT = CoralDevExperiment()` at module level — replay.py imports this instance without needing to instantiate with arguments.

### 3. Generic framework replay (`coralai/replay.py` + `coralai/replay_utils.py`)

**Before**: `experiments/coral_dev/replay.py` hard-coded the channel layout and couldn't replay other experiments.

**After**: `coralai/replay.py` is experiment-agnostic. It:
1. Reads `run_dir/snapshot/experiment.py` via `importlib`
2. Gets `EXPERIMENT` instance
3. Calls `exp.make_substrate()`, `exp.make_evolver()`, `evolver.load_checkpoint()`
4. Runs the physics loop using `exp.run_physics()` / `exp.run_evolution()`

`coralai/replay_utils.py` splits out `discover_checkpoints()` and `load_experiment_from_snapshot()` as importable-without-Taichi utilities (used by tests and replay.py).

**`GenericReplayVis`** GUI panel layout:
- **Replay panel**: run name, checkpoint counter, Reset/Prev/Next buttons, Paused checkbox
- **Display panel**: norm controls, R/G/B channel sliders
- **Stats panel**: cursor readout, step, energy_offset, genome leaderboard (top 6)

### 4. `run.py` slimmed to a driver (`experiments/coral_dev/run.py`)

Removed: `CHANNELS`, `KERNEL`, `DIR_ORDER`, `SENSE_CHS`, `ACT_CHS`, `CONFIG_PATH` constants. Physics and evolution logic moved to `experiment.py`.

Added:
- `--resume-from CKPT_DIR` — load checkpoint, continue from that step; `--steps` counts additional steps
- `--env` / `--env-param` — selects StartEnvironment subclass
- Step-0 checkpoint saved on fresh runs so replay can always start from the beginning
- `meta.json` includes `env`, `env_persist`, `start_step`, `resumed_from`
- `initial_state.pt` saved once at run start (not per checkpoint)

Step loop delegates:
```python
exp.run_physics(substrate, evolver)
exp.run_evolution(substrate, evolver, step)
if env.persist:
    env.step(substrate)
```

## Key implementation bugs hit

### `inspect.getfile()` on dynamic modules
`torch` patches `inspect.getfile` globally and raises `TypeError: is a built-in class` for any class loaded via `importlib.spec_from_file_location`. Solution: capture `_exp_dir` at module definition time, not at call time.

### Write tool corruption
The Write tool produced a partial corrupted `run.py` on first write (claimed success but the file had ~52 lines changed while leaving old `CHANNELS`/`env_mask` code intact). Discovered via `git diff`. Had to completely rewrite.

### smoke_test ambiguous run dir
First version used `latest_run_dir()` which could find snapshot subdirs newer than the actual run dir. Fixed by: regex-filtering `runs/coral_dev_*` for `\d{8}_\d{6}$`, and having each test return and pass its `run_dir` explicitly.

## Non-determinism investigation

Tested reproducibility by resuming from the same checkpoint twice. Result: DIFFERS on both MPS and CPU backends.

**Two sources:**
1. **Taichi atomic race conditions** (`explore_physics`): Multiple cells compete to colonize the same neighbor via `ti.atomic_*`. Metal doesn't guarantee thread execution order. Same input state → different thread scheduling → different colonization outcomes → n_genomes diverges discretely (not just float accumulation). This is the main cause.
2. **MPS float non-determinism** (~0.01%): Parallel reductions (sum, matmul) may accumulate in different order across runs.

**All RNG states are saved in checkpoints** (torch CPU, MPS, numpy, python) and correctly restored on load. The divergence is in the Taichi kernel execution order itself, not the random number streams.

For bitwise determinism: would require serializing the atomic ops in `explore_physics` (e.g. sequential cell ordering), which would be a significant performance hit. Not pursued — checkpoints provide enough experimental reproducibility for the research use case.
